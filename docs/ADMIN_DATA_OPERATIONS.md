# 管理者向け 参加者登録・削除・開催データ初期化

版: v1.1
作成日: 2026-07-17（JST）

## できること

管理画面では、次の3つのデータ操作を行えます。

1. 参加者を1人ずつ追加
2. 参加者名とDiscord User IDの一覧から最大100人をまとめて追加
3. 参加者の利用停止、または次回開催へ向けた開催データの初期化

Discord User IDは、送信前と通信失敗時だけ入力欄へ一時的に保持します。APIで照合用HMACへ変換した後に破棄し、D1、操作履歴、ブラウザの保存領域へIDそのものを保存しません。

## 参加者をまとめて追加する

1. 「参加者名」へ、Aさん、Bさん、Cさんのように1人ずつ改行して入力する。見出しは入れない。
2. 「DiscordユーザーID」へ、同じ人の順番で1人ずつ改行して入力する。
3. 全員に適用するグループと権限を選ぶ。グループや権限が違う人は分けて追加する。
4. 2か所の人数が同じことを確認し、「名前とIDの組み合わせを確認」を押す。
5. 「1人目」のように表示される名前とID末尾4桁の組み合わせを確認する。
6. 「この○人をまとめて追加」を押す。

カンマ、空白、タブ、見出し行で区切る必要はありません。必ず1人ごとに改行してください。名前とDiscord User IDの先頭・途中に空行がある場合や、2か所の人数が違う場合は、別人のIDを結び付けないため追加を止めます。

権限は`一般参加者`または`担当者`です。標準は`一般参加者`で、管理者は一括追加できません。1人でも入力不備または登録済みDiscordアカウントがある場合は、全員を追加せずに止まります。入力を直して同じ一覧を再送してください。

## 参加者を削除する

「参加者から削除」は、安全な利用停止です。

- 参加者一覧から外れる。
- Discordのログイン許可を消す。
- 現在ログイン中の端末をログアウトさせる。
- 過去の注文、会計、操作履歴は残す。
- 唯一の管理者は削除できない。

誤って停止した場合は、履歴を消さずに参加者を登録し直せます。本人確認済みのDiscord User IDをもう一度登録してください。

## 開催データを初期化する

この操作は次回開催の準備用です。D1全体を消す操作ではありません。

削除するもの:

- 全注文と取消記録
- 管理者以外の参加者データ
- 現在操作中の管理者sessionを除くログインsession
- 未使用のOAuth一時状態

残すもの:

- 唯一の管理者と、そのDiscord照合用HMAC
- 現在操作中の管理者session
- メニュー
- 操作履歴と初期化の監査記録（削除した参加者・担当者との紐づけは解除され、実行者は「システム」表示になる）
- schema、migration情報、ID採番履歴

### productionでの実行条件

1. 参加者へ受付停止を案内する。
2. Cloudflare D1のTime Travel bookmarkを取得する。
3. bookmarkと取得時刻を、秘密情報を含まない非公開release記録へ保存する。
4. 管理画面からログアウトし、Discordでログインし直す。
5. 「安全な操作」で削除件数と残る件数を確認する。
6. 復元地点を記録済みの確認欄を選ぶ。
7. 指定された確認文を入力し、実行する。

画面の件数が途中で変わった場合や、ログインから5分を過ぎた場合は実行を止めます。画面を更新するか、ログインし直してください。

## 実行後の確認

- 注文が0件
- activeな管理者が1人
- メニュー件数が実行前と同じ
- 操作履歴に初期化が1件記録されている
- D1の`PRAGMA foreign_key_check`が0件
- 未登録Discordアカウントがログイン拒否される

production初回実行の前に、必ずstagingの架空データで初期化と復元の両方を確認します。本番D1の初期化や復元は、ローカル実装の完了だけでは許可されません。
